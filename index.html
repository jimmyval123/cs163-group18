// OVERVIEW

Project Summary

Motivation

League of Legends is one of the world's most-played games, yet winning remains poorly understood at a statistical level. Conventional wisdom says "get kills to win" — our analysis shows this is far too simplistic. Success is champion-specific, not universally defined.

Approach

We built individual logistic regression models for each champion using standardized performance features. Coefficient vectors were then clustered via K-Means and visualized with UMAP and PCA to reveal structural groupings across the champion pool.

Key Finding: Champions like Jinx (Marksman) win through KDA and damage dealing, while supports like Braum win through crowd control and mitigation — not kills. Our models capture these role-specific win conditions with high accuracy (AUC ≈ 0.8+).

// RESEARCH QUESTIONS

Preliminary Results

Q1 — Performance Statistics & Win Probability

We found that champions had varying levels of win correlation for each performance statistic based on their role and playstyle. For example, Jinx (an attack damage carry) has win correlation skewed towards offensive stats like kills and damage. Braum (a support) shows little positive correlation with those same stats — his value comes from crowd control and protecting teammates.

Implication: Being successful is champion-specific rather than universally defined. Players and coaches can use these findings to set champion-specific benchmarks for performance evaluation instead of relying on a universal standard like "more kills = higher chance to win."

Jinx — Standardized logistic regression coefficients. KDA and total damage are the strongest predictors of her winning.

Q2 — Clustering Champions by Win Profile

Non-support champions were clustered by similarities in their win profiles. Since champion win profiles are highly unique, the data resisted clean separation. Multiple feature selection methods were used (forward/backward selection, genetic selection, random combinations) to find the best data separation. Some clustering emerged but was not definitively clean.

Implication: Champions were broadly groupable by playstyle, but uniqueness of each win profile means further feature exploration is needed. The current results reveal how polarized each champion's win profile is — a feature, not a bug.

UMAP projection of all champions colored by cluster. Diamond markers (♦) indicate support class; circle markers (●) indicate carry class.

Q3 — Clusters vs. Official Champion Archetypes

Identified clusters partially align with existing archetypes. With four clusters, carry champions split into frontline vs. backline, and support champions do the same. The partial alignment is actually strong evidence the analysis is reliable — complete departure from official classifications would suggest nonsensical results.

Notable Finding: Some Juggernauts (typically beefy frontliners) appear in the backline cluster. This suggests those champions can be played optimally from greater distance than previously assumed — opening creative team composition options.

// EXPLORATORY DATA ANALYSIS

Data Overview

Data Source

Match data was collected via the Riot Games API, covering ranked games across multiple patches. Each row represents a single player's performance in one match, labeled with win/loss outcome.

Features Used

Thirteen performance statistics were selected: KDA, total damage to champions, assists, time spent dead, champion level, wards placed, neutral minions killed, total CC dealt, mitigation ratio, vision score, time CC-ing others, longest time spent living, and CS (creep score).

// ANALYTICAL METHODS

Modeling Approach

Per-Champion Logistic Regression

A separate logistic regression model was trained for each champion with sufficient game data. Features were standardized (z-score) before fitting, making coefficients directly comparable across features. The binary target is win/loss.

Evaluation Metric

Each model is evaluated using AUC-ROC score, which measures ability to discriminate wins from losses regardless of class balance. Most champion models achieve AUC ≈ 0.75–0.90, indicating strong predictive power.

Clustering Pipeline

Coefficient vectors from each champion's model were assembled into a matrix. Feature selection methods (forward selection, backward elimination, genetic algorithms) identified the most informative subset of coefficient dimensions for clustering. K-Means was applied with k=5, followed by UMAP for 2D visualization.

Finding 1 — Win Conditions Are Champion-Specific

There is no single universal set of statistics that predicts winning. Each champion has a distinct "win profile" shaped by their role and kit. Using a one-size-fits-all metric (e.g., kills) to evaluate performance is fundamentally flawed.

Finding 2 — Clusters Broadly Reflect Playstyle

Even though clean separation proved difficult, champions did cluster into identifiable groups. Carries split into frontline and backline types. Supports clustered separately. This validates that the coefficient-based approach captures meaningful role structure.

Finding 3 — Partial Archetype Alignment Validates the Method

The fact that clusters partially (but not completely) align with official Riot classifications is meaningful. Complete agreement would suggest we simply re-discovered known labels. Disagreement in certain cases (e.g., Juggernauts in the backline cluster) points to novel insights about non-standard playstyles.

// FUTURE WORK

Limitations & Next Steps

Confounding: Game State

Stats like deaths may reflect the team being already behind, not be the cause of losing. Future models should incorporate game-state variables (gold differential, objective control) to better isolate causal effects.

Feature Exploration

Clustering quality can be improved with richer features — e.g., itemization patterns, positional data, patch version — to better separate champion win profiles into more definitive groups.

Practical Application: Players and coaches can use champion-specific benchmarks to set meaningful performance targets. For example, a Jinx player should prioritize KDA and damage output, while a Braum player should focus on crowd control uptime and vision score — not kills.
